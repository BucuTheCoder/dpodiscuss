DPO Web Services

Internship Project in Saint Louis University - Data Protection Office

About The Project

The project, titled "Web Services for Data Protection Office," is an internship initiative focused on replacing the current use of Google Forms with a more robust web application for managing sensitive data. This new web app is designed to streamline the submission of incident reports, requests for data disposal, document reviews, and requests for student personal data. By transitioning from Google Forms to a dedicated web application, the project aims to enhance the efficiency, security, and user experience of data management within the institution. It reduces the manual workload on staff, ensuring that data protection protocols are consistently followed. The web app provides a user-friendly interface that simplifies interaction for both staff and students while ensuring compliance with data protection regulations. Additionally, the project safeguards sensitive information, aligning with current data protection standards. My Role

In this project, I served as a full stack developer, responsible for both the backend and frontend development. I utilized PHP to build the backend infrastructure, ensuring the web services are robust, secure, and efficient. On the frontend, I employed the Bootstrap framework to create a responsive and user-friendly interface. My work involved integrating these components to provide a seamless experience for users submitting incident reports, data disposal requests, document reviews, and student personal data requests. This comprehensive approach allowed me to contribute to the overall functionality and design of the web application, ensuring it meets the needs of the Data Protection Office.

Features of the Project

  • Incident Report Submission: Users can securely submit reports of data-related incidents.
  • Request for Data Disposal: A feature for requesting the secure disposal of sensitive data.
  • Document Review Requests: Enables users to request reviews of important documents for compliance.
  • Student Personal Data Requests: Allows users to request access to or modification of student personal data.
  • User Authentication: Secure login system to protect sensitive data and ensure authorized access.
  • Responsive Design: Frontend is built with Bootstrap to ensure the application is accessible on various devices.
  • Admin Dashboard: Provides a centralized interface for managing and tracking all submissions and requests.
